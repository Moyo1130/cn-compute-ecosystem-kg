# 知识图谱实体与关系设计规范-v12.0

## 1. 实体定义

### 1.1 软件（Software）- 抽象层
- **描述**：抽象的软件或模型概念，不依赖具体框架
- **属性**：
  - id：唯一标识符，"software"和name字段值拼接，删除空格，大写转小写，用":"作连接符（例如：software:acnet）
  - label：Software
  - name : 软件/模型名称（vLLM、LLaMA2、YOLOv8）
  - source_url : - 代码仓库地址去掉最后一层框架路径（https://gitee.com/deep-spark/deepsparkhub/tree/master/models/audio/speech_recognition/conformer_wenet）
  - extra/Type : 类型（模型）
  - extra/area : 研究领域（cv、nlp）
  - extra/Main-Task : 主要任务（文本生成、目标检测等）

### 1.2 框架（Framework）
- **描述**：深度学习框架
- **属性**：
  
  - id：唯一标识符，"framework"和name字段值拼接，删除空格，大写转小写，用":"作连接符（例如：framework:pytorch）
  - label：Framework
  - name : 框架名称（PyTorch、TensorFlow、Paddlepaddle、Mindspore）
  
  - source_url : 框架官网

### 1.3 软件实例（SoftwareInstance）- 具体层
- **描述**：特定框架下的软件实现，由软件和框架唯一确定
- **属性**：
  - id：唯一标识符，"softwareInstance"和name字段值拼接，删除空格，大写转小写，用":"作连接符。（例如：softwareInstance:acnet_pytorch）
  - label：SoftwareInstance
  - software_id：代码仓库地址(https://gitee.com/deep-spark/deepsparkhub/tree/master/models/audio/speech_recognition/conformer_wenet/pytorch)
  - framework_id：对应框架的ID
  - name：格式：`{software_name}_{framework_name}`（例如：acnet_Pytorch）

### 1.4 部署配置（DeploymentConfig）- 硬件-运行时组合
- **描述**：硬件与运行时的组合配置，独立于具体软件
- **属性**：
  - id：唯一标识符，"deploymentconfig"和name字段值拼接，删除空格，大写转小写，用":"作连接符。（例如：deploymentconfig:bI-v150_ixucasdk430）
  - label：DeploymentConfig
  - hardware_id：硬件ID
  - runtime_id：运行时ID
  - name：格式：`{hardware_name}_{runtime_name}`（例如：BI-V150_IXUCA SDK 4.3.0）

### 1.5 硬件（Hardware）
- **描述**：国产GPU硬件设备
- **属性**：
  - id：唯一标识符，"hardware"和name字段值拼接，删除空格，大写转小写，用":"作连接符（例如：hardware:bi-v100）
  - label：Hardware
  - name : 国产 GPU型号（天数BI-V100、BI-V150）
  - extra/Vendor : 厂商信息（天数智芯）

### 1.6 运行时（Runtime）
- **描述**：硬件对应的运行时环境，提供API接口
- **属性**：
  - id：唯一标识符，"runtime"和name字段值拼接，删除空格，大写转小写，用":"作连接符（例如：runtime:ixucasdk310）
  - label：Runtime
  - name : 对应品牌和版本的运行时（IXUCA SDK 3.1.0）
  - extra/release : 运行时发布日期（23.09）

### 1.7 数据集（Dataset）
- **描述**：用于训练或评估的数据集
- **属性**：
  - id：唯一标识符，"dataset"和name字段值拼接，删除空格，大写转小写，用":"作连接符（例如：dataset:coco）
  - label：Dataset
  - name：数据集名称（COCO、ImageNet、CLUE等）

### 1.8 教程文档（Guide）
- **描述**：软件相关的教程、文档、指南
- **属性**：
  - id：唯一标识符，"guide"和name字段值拼接，删除空格，大写转小写，用":"作连接符（例如：dataset:coco）
  - label：Guide
  - name：文档名称
  - source_url：文档链接

### 1.9 作者（Author）
- **描述**：开发者或个人、机构
- **属性**：
  - id：唯一标识符，"author"和name字段值拼接，删除空格，大写转小写，用":"作连接符（例如：author:sanghoon）
  - label：Author
  - name：开发者/机构名称
  - source_url：GitHub/Gitee链接

### 1.10 软件栈接口（API）
- **描述**：运行时提供的编程接口
- **属性**：
  - id：唯一标识符，"API"和name字段值拼接，删除空格，大写转小写，用":"作连接符（例如：API:cublasCreate）
  - label：API
  - name：接口名称（cublasCreate、cublasDestroy）
  - source_url : 软件栈文档链接
  - extra/library : 接口所属软件库/标准（ixBLAS）

## 2. 关系定义

### 2.1 核心关系

| 关系类型 | 起点 → 终点 | 基数 | 描述 |
|---------|------------|------|------|
| **INSTANCE_OF** | SoftwareInstance → Software | 1:N | 软件实例属于哪个抽象软件 |
| **USES_FRAMEWORK** | SoftwareInstance → Framework | 1:N | 软件实例使用哪个框架 |
| **TRAINED_ON** | SoftwareInstance → Dataset | N:M | 软件实例在哪个数据集上训练 |
| **HAS_GUIDE** | SoftwareInstance → Guide | 1:1 | 软件实例对应哪个文档 |
| **DEVELOPED_BY** | Software → Author | N:M | 软件由哪些开发者/机构开发 |
| **SUPPORTS_RUNTIME** | Hardware → Runtime | N:M | 硬件支持哪些运行时 |
| **SUPPORTS_API** | Runtime → API | N:M | 运行时支持哪些API接口 |
| **CONFIGURES_HARDWARE** | DeploymentConfig → Hardware | 1:N | 部署配置关联的硬件 |
| **CONFIGURES_RUNTIME** | DeploymentConfig → Runtime | 1:N | 部署配置关联的运行时 |
| **CAN_DEPLOY_ON** | SoftwareInstance → DeploymentConfig | N:M | 软件实例可以在哪些部署配置上运行 |
| **MAPS_TO** | API → API | 1:1 | CUDA API对应的HIP API是哪个 |
| **COMPATIBLE_WITH** | Runtime → API | N:M | 运行时兼容哪些API接口 |
